# Akasha Repository Classes

This document defines the canonical classes of repositories that may
exist within the Akasha ecosystem.

Each class represents a structural role within the system.

Repositories should declare their class in `repo-manifest.yaml`.

---

# Foundation Repositories

These define the laws and structure of the ecosystem.

## law

Defines governance and principles.

Example:

- akasha-axioms

Responsibilities:

- ecosystem laws
- governance doctrine
- discovery doctrine
- tool creation doctrine

---

## ontology

Defines structural standards and schemas.

Example:

- akasha-world

Responsibilities:

- naming conventions
- repository schemas
- structural templates

---

## registry

Tracks ecosystem membership.

Example:

- akasha-constellation

Responsibilities:

- repository registry
- ecosystem mapping
- dependency graph

---

# Governance Repositories

These maintain ecosystem integrity.

## guardian

Detects structural drift and alignment issues.

Example:

- akasha-guardian

Responsibilities:

- repo validation
- manifest checks
- naming compliance
- drift reporting

---

# Creation Repositories

These generate or evolve tools.

## forge

Creates tools and repository scaffolds.

Example:

- akasha-forge

Responsibilities:

- repository creation
- tool generation
- template expansion
- structural repair proposals

---

# Discovery Repositories

These explore unknown design space.

## discovery-engine

Runs discovery experiments.

Example:

- akasha-discovery

Responsibilities:

- hypothesis generation
- structural exploration
- candidate discovery

---

# Analysis Repositories

These interpret artifacts and results.

## lens

Provides analytical perspective.

Example:

- akasha-lens

Responsibilities:

- semantic interpretation
- artifact analysis
- pattern recognition

---

# Execution Repositories

These run experiments or tasks.

## observer

Executes experiments and collects outputs.

Example:

- akasha-observer

Responsibilities:

- experiment execution
- runtime evaluation
- result capture

---

# Memory Repositories

These preserve knowledge.

## record

Stores long-term system memory.

Example:

- akasha-record

Responsibilities:

- discovery logs
- experiment archives
- artifact preservation

---

# Graph Repositories

These contain structured knowledge graphs.

## domain-graph

Represents a graph for a specific domain.

Examples may include:

- phase behavior graphs
- chemistry graphs
- mathematics graphs

Responsibilities:

- structured node/edge storage
- domain relationship modeling

---

# Guiding Principle

Repositories should belong to a known class.

If a new class is needed, it must be defined here before repositories
of that type are created.

This ensures that the Akasha ecosystem remains structured and
understandable as it grows.
